Web Design Skill is a drop-in skill file — there is no package to install, no CLI to configure, and no account to create. You copy two files into your project, and your agent picks them up automatically the next time you make a front-end request.

## File structure

Create the following directory structure inside your project:

```text
your-project/
├── .agents/skills/web-design-engineer/
│   ├── SKILL.md
│   └── references/
│       └── advanced-patterns.md
└── ...
```

<Note>
Some tools use `.claude/skills/` instead of `.agents/skills/`. Place the files in whichever directory your agent expects. The file content is identical regardless of location.
</Note>

## Installation steps

<Steps>
  <Step title="Copy the skill files">
    Clone or download the [web-design-skill repository](https://github.com/ConardLi/web-design-skill) and copy the two skill files into your project:

    ```bash
    # Clone the repository
    git clone https://github.com/ConardLi/web-design-skill.git

    # Copy the skill directory into your project
    cp -r web-design-skill/.agents your-project/
    ```

    Your project should now contain:

    ```text
    your-project/
    ├── .agents/skills/web-design-engineer/
    │   ├── SKILL.md
    │   └── references/
    │       └── advanced-patterns.md
    ```
  </Step>

  <Step title="Start your agent">
    Open your project in Claude Code, Cursor, or any SKILL.md-compatible agent. The agent will read the skill file automatically when you make a request that involves visual or interactive front-end work.

    No additional configuration is required. The skill activates based on the nature of your request — you do not need to reference the file explicitly.
  </Step>

  <Step title="Request an artifact">
    Write a prompt that describes the visual or interactive output you want. The skill applies whenever your intent is to produce something visual, interactive, or presentational — even if you don't use the words "HTML" or "web page."

    Example prompt:

    ```text
    Build a homepage for an independent photographer's portfolio. The feel should be
    editorial and minimal — restrained palette, strong typography, generous whitespace.
    Include a hero image area, a selected works grid, an about section, and a contact footer.
    ```

    The agent will recognize this as a front-end deliverable and activate the Web Design Skill workflow.
  </Step>

  <Step title="Approve the design declaration">
    Before writing any code, the agent outputs a design system declaration in Markdown for you to review. This is Step 3 of the six-step workflow — the agent articulates its color, typography, spacing, and motion choices before committing to them in code.

    A typical declaration looks like:

    ```markdown
    Design Decisions:
    - Color palette: Paper-warm light (#f2efe8) primary background, ink-dark (#161513) text,
      no accent color — extremely restrained two-tone palette
    - Typography: Instrument Serif (display headings) + Space Grotesk (UI, captions)
    - Spacing system: 8px base unit, major sections at 120px vertical rhythm
    - Border-radius strategy: sharp (0–4px) — avoids the large-radius card cliché
    - Shadow hierarchy: none — depth created through color contrast and whitespace alone
    - Motion style: slow, film-like — Ken Burns on hero (24s), 600ms ease-out entry animations
    ```

    Review the declaration and confirm to proceed, or give feedback to adjust the direction before a line of code is written. This is the most efficient moment to course-correct.
  </Step>
</Steps>

## What output types the skill covers

The skill applies to any visual or interactive front-end deliverable:

| Output type | Examples |
|---|---|
| Web pages and landing pages | Marketing sites, product pages, portfolios |
| Interactive prototypes | Clickable app mockups with device frames (iPhone, browser window) |
| HTML slide decks | 1920×1080 presentations with keyboard navigation |
| Data visualizations | Dashboards built with Chart.js or D3.js |
| Animations | CSS/JS motion design, timeline-driven demos with play/pause scrubber |
| Design systems | Token exploration, component variants, side-by-side design canvases |

<Tip>
If your request involves something visual — even if it's a quick throwaway prototype or a design comparison — the skill will engage. You do not need to explicitly mention HTML or the skill name.
</Tip>
